# 02. Авторизация и права доступа

## Авторизация

Авторизация выполняется по Telegram ID пользователя.

Первый администратор создается техническим bootstrap при старте приложения:

- приложение читает переменную окружения `ADMIN_TELEGRAM_IDS`;
- значение содержит один или несколько Telegram ID администраторов;
- для отсутствующих пользователей из списка создаются или активируются записи с ролью `admin`.

Алгоритм:

1. Пользователь отправляет `/start` или любое сообщение боту.
2. Бот получает Telegram ID.
3. Бот ищет активного пользователя с этим Telegram ID.
4. Если пользователь найден и не заблокирован, бот показывает меню по роли.
5. Если пользователь заблокирован, бот показывает отказ.
6. Если пользователь не найден, бот разрешает только сценарий подачи заявки на доступ.

Текст отказа:

```text
Доступ запрещён. Ваш аккаунт не зарегистрирован в системе.
```

## Заявка бригадира на доступ

Неизвестный Telegram ID не получает доступ к функциям системы, но может подать заявку:

1. Пользователь нажимает `/start`.
2. Бот просит ввести имя и фамилию одним сообщением.
3. Пользователь отправляет имя и фамилию.
4. Бот создает заявку со статусом `pending`.
5. До решения администратора пользователь может видеть только сообщение об ожидании.

Администратор видит список заявок и может:

- принять заявку;
- отклонить заявку.

При принятии заявки бот создает пользователя с ролью `foreman`.
При отклонении пользователь не получает доступ к функциям системы.

## Роли

### Администратор

Администратор управляет всеми объектами, пользователями, расходами и отчетами.

### Бригадир

Бригадир работает только с назначенными ему активными объектами.

## Блокировка пользователя

Заблокированный пользователь не получает доступ к системе, даже если его Telegram ID есть в базе.

При блокировке:

- пользователь остается в базе;
- его история расходов сохраняется;
- новые действия в боте запрещены.

## Удаление пользователя

Удаление бригадира в MVP трактуется как мягкое удаление:

- ставится `deleted_at`;
- пользователь больше не авторизуется;
- история расходов сохраняется;
- активные назначения на объекты закрываются.
