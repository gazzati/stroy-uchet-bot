# 03. Доменная модель и бизнес-правила

## Пользователь

Пользователь - человек, которому разрешен доступ к боту.

Типы пользователей:

- администратор;
- бригадир.

Бизнес-правила:

- `telegram_id` должен быть уникальным среди неудаленных пользователей;
- пользователь без роли не должен получать доступ к системе;
- заблокированный пользователь не может выполнять действия;
- удаление пользователя не удаляет его расходы.

## Объект

Объект - строительный объект, по которому ведется учет расходов.

Статусы:

- `active` - объект доступен для работы;
- `archived` - объект скрыт у бригадиров, новые расходы запрещены.

Бизнес-правила:

- у объекта должно быть название;
- бюджет должен быть неотрицательным;
- объект можно архивировать только из статуса `active`;
- объект можно восстановить только из статуса `archived`;
- архивирование не удаляет расходы и назначения.

## Назначение бригадира

Назначение связывает бригадира с объектом.

Правило MVP:

- один бригадир может быть назначен на несколько объектов;
- один объект может иметь несколько бригадиров.

Бизнес-правила:

- назначать можно только активного и незаблокированного бригадира;
- назначать можно на активный объект;
- повторное активное назначение одного бригадира на один объект запрещено;
- открепление закрывает назначение через `deleted_at`.

## Расход

Расход - финансовая операция по объекту.

Типы:

- `manual_purchase` - ручная закупка;
- `document_purchase` - чек или накладная;
- `bank_transfer` - безналичная операция;
- `service_payment` - оплата услуг.

Общие правила:

- расход всегда относится к одному объекту;
- расход всегда имеет автора;
- сумма должна быть больше нуля;
- расход нельзя добавить по архивному объекту;
- удаление расхода мягкое через `deleted_at`;
- удаленный расход не участвует в расчете бюджета и отчетах по умолчанию;
- расход сразу учитывается в бюджете после создания, без подтверждения администратором;
- администратор может создавать расходы по любому активному объекту;
- бригадир может создавать расходы только по назначенным активным объектам;
- администратор может редактировать любые неудаленные расходы;
- бригадир может редактировать только свои расходы по активным объектам;
- администратор может мягко удалять любые расходы;
- бригадир может мягко удалять только свои расходы по активным объектам;
- временного ограничения на редактирование и удаление своих расходов бригадиром в MVP нет.

## Поля по типам расходов

### Ручная закупка

Обязательные поля:

- объект;
- наименование;
- количество;
- единица измерения;
- цена за единицу.

Опциональные поля:

- комментарий.

Запрещенные поля:

- фото документа.

Единица измерения вводится свободно. Бот предлагает быстрые варианты:

- `шт`;
- `м`;
- `м2`;
- `м3`;
- `кг`;
- `л`;
- `уп`;
- `меш`.

Расчет:

```text
amount = round(quantity * unit_price)
```

`quantity` может быть дробным. `unit_price` хранится в копейках. Если произведение дает дробную копейку, сумма округляется до ближайшей копейки.

### Чек / накладная

Обязательные поля:

- объект;
- наименование;
- сумма.

Опциональные поля:

- фото документа;
- комментарий.

Запрещенные поля:

- количество;
- единица измерения;
- цена за единицу.

### Безналичная операция

Обязательные поля:

- объект;
- наименование;
- сумма.

Опциональные поля:

- комментарий.

Запрещенные поля:

- количество;
- единица измерения;
- цена за единицу;
- фото документа.

### Оплата услуг

Обязательные поля:

- объект;
- наименование;
- сумма.

Опциональные поля:

- комментарий.

Запрещенные поля:

- количество;
- единица измерения;
- цена за единицу;
- фото документа.

## Бюджет

Показатели:

- бюджет объекта;
- сумма расходов;
- остаток.

Формула:

```text
Остаток = Бюджет - Сумма неудаленных расходов объекта
```

Правило MVP:

- превышение бюджета не блокирует добавление расходов;
- бот показывает отрицательный остаток, если расходы превысили бюджет.

## Денежные значения

Решение MVP:

- валюта - рубли;
- хранение в базе - целые копейки;
- пользовательский ввод допускает рубли и копейки;
- отображение пользователю - рубли с двумя знаками после запятой или без копеек, если копейки равны нулю.
