# 05. Сценарии Telegram-бота

## Общие правила интерфейса

- После `/start` бот показывает главное меню согласно роли.
- Если пользователь не зарегистрирован, `/start` запускает только подачу заявки на доступ бригадира.
- Любое действие должно проверять актуальные права доступа.
- Если пользователь потерял доступ во время сценария, бот отменяет сценарий и показывает отказ.
- Если объект стал архивным во время добавления расхода, бот запрещает сохранение.
- Для длинных списков нужна пагинация.
- Денежный ввод принимает рубли и копейки: `1250`, `1250.50`, `1250,50`.
- Все сообщения, меню и отчеты MVP на русском языке.

## Главное меню администратора

- `🏗 Объекты`
- `👷 Бригадиры`
- `📝 Заявки`
- `📊 Отчеты`

## Главное меню бригадира

- `🏗 Мои объекты`
- `➕ Добавить расход`
- `📋 Мои расходы`

## Администратор: управление объектами

### Список объектов

Путь:

```text
🏗 Объекты -> Активные объекты
```

Бот показывает:

- название объекта;
- бюджет;
- текущие расходы;
- остаток;
- кнопки действий.

Действия:

- создать объект;
- открыть объект;
- архивные объекты.

### Создание объекта

Шаги:

1. Администратор нажимает `Создать объект`.
2. Бот запрашивает название.
3. Бот запрашивает бюджет.
4. Бот создает объект в статусе `active`.
5. Бот показывает карточку объекта.

Валидация:

- название не пустое;
- бюджет неотрицательный;
- бюджет должен быть числом.

### Карточка объекта

Показывает:

- название;
- статус;
- бюджет;
- расходы;
- остаток;
- список назначенных бригадиров.

Действия:

- переименовать;
- изменить бюджет;
- назначить бригадира;
- открепить бригадира;
- добавить расход;
- расходы объекта;
- архивировать;
- назад.

### Архивирование объекта

Шаги:

1. Администратор выбирает активный объект.
2. Нажимает `Архивировать`.
3. Бот просит подтверждение.
4. После подтверждения объект получает статус `archived`.

После архивации:

- объект исчезает из списков бригадиров;
- добавление расходов запрещено;
- отчеты доступны администратору.

## Администратор: управление бригадирами

### Заявки бригадиров

Путь:

```text
📝 Заявки
```

Бот показывает pending-заявки:

- имя и фамилию;
- Telegram ID;
- username, если есть;
- статус заявки.

Действия:

- принять заявку;
- отклонить заявку;
- вернуться в меню.

При принятии заявки бот:

1. Создает пользователя с ролью `foreman`.
2. Помечает заявку как `accepted`.
3. Уведомляет бригадира, что доступ открыт.
4. Показывает администратору карточку созданного бригадира.

При отклонении заявки бот:

1. Помечает заявку как `declined`.
2. Уведомляет заявителя об отклонении.
3. Возвращает администратора к списку заявок.

### Список бригадиров

Путь:

```text
👷 Бригадиры
```

Бот показывает:

- имя;
- Telegram ID;
- статус доступа;
- количество активных назначений.

Действия:

- открыть карточку бригадира.

### Карточка бригадира

Показывает:

- имя;
- Telegram ID;
- username, если есть;
- статус блокировки;
- активные объекты.

Действия:

- изменить имя;
- заблокировать или разблокировать;
- назначить на объект;
- открепить от объекта;
- удалить.

## Бригадир: мои объекты

Путь:

```text
🏗 Мои объекты
```

Бот показывает только активные объекты, на которые назначен бригадир.

Карточка объекта для бригадира показывает:

- название;
- бюджет;
- расходы;
- остаток;
- кнопку добавления расхода;
- кнопку просмотра расходов.

## Добавление расхода

Сценарий доступен:

- администратору для любого активного объекта;
- бригадиру только для назначенного активного объекта.

Общие шаги:

1. Пользователь выбирает объект.
2. Пользователь выбирает тип расхода.
3. Бот запрашивает поля выбранного типа.
4. Бот показывает экран подтверждения.
5. Пользователь подтверждает.
6. Бот сохраняет расход.
7. Бот показывает обновленные расходы и остаток объекта.
8. Если остаток стал отрицательным, бот показывает предупреждение о превышении бюджета.

### Ручная закупка

Шаги:

1. Наименование.
2. Количество.
3. Единица измерения: быстрые варианты `шт`, `м`, `м2`, `м3`, `кг`, `л`, `уп`, `меш` или свободный ввод.
4. Цена за единицу.
5. Бот рассчитывает сумму как `round(количество * цена за единицу)` до ближайшей копейки.
6. Комментарий или пропуск.
7. Подтверждение.

### Чек / накладная

Шаги:

1. Наименование.
2. Фото документа или пропуск.
3. Сумма.
4. Комментарий или пропуск.
5. Подтверждение.

### Безналичная операция

Шаги:

1. Наименование.
2. Сумма.
3. Комментарий или пропуск.
4. Подтверждение.

### Оплата услуг

Шаги:

1. Наименование.
2. Сумма.
3. Комментарий или пропуск.
4. Подтверждение.

## Просмотр расходов

Для администратора:

- расходы по объекту;
- расходы по бригадиру.

Для бригадира:

- свои расходы;
- все расходы по назначенному объекту.

Правило MVP:

- в разделе `📋 Мои расходы` бригадир видит только свои расходы.
- в карточке назначенного объекта бригадир видит все расходы этого объекта, включая расходы других бригадиров.

## Редактирование и удаление расходов

Доступ:

- администратор может редактировать и мягко удалять любой неудаленный расход;
- бригадир может редактировать и мягко удалять только свои расходы по активным объектам;
- временного ограничения на редактирование и удаление своих расходов бригадиром в MVP нет;
- расходы по архивному объекту нельзя редактировать бригадиру, даже если расход его.

После мягкого удаления:

- расход скрывается из обычных списков;
- расход не участвует в расчете бюджета;
- расход не попадает в CSV по умолчанию.
